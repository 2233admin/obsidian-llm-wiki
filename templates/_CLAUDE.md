## Auto-Save Rules

Claude should auto-save **without asking**:
- Decisions made in conversation -> relevant project note + daily note
- New research findings -> research folder
- Engineering learnings -> engineering folder
- Project status changes -> project notes

Claude should **ask before saving**:
- Anything involving deleting or archiving
- Financial data or credentials
- Content in folders marked "do not modify"

## Propagation Rules

| Event | Also update |
|-------|-------------|
| New project started | Project note + dashboard + daily note |
| Research completed | Research note + related project + daily note |
| Decision made | Project note (Key Decisions section) + daily note |

**Never create a note in isolation.** Every write ripples to the daily note and linked notes.

## Search Before Create

Before creating any note:
1. Search vault for existing note with similar title/topic
2. If found and same concept -> update existing, don't create new
3. Duplicate notes are vault rot
