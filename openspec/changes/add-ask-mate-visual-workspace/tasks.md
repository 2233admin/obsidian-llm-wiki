## Delivery contract

The change ships as eight end-to-end tracer bullets rather than 63 independently scheduled implementation fragments. Each bullet must produce one user-visible result, deterministic fixtures, at least one failure-closed scenario, and the relevant MCP/CLI/Obsidian parity evidence. Documentation, privacy, license, migration, and release gates travel with the slice they verify.

## 1. Vocabulary and executable contracts

- [x] Add the canonical Visual Workspace, Mind Map Document, Visual Edit Plan, Graph Relation Evidence, Problem Intake, Problem Observation, Issue Change Plan, Problem Disposition, and External Contribution Plan vocabulary; create shared Visual Workspace and Problem Intake packages; extend the shared graph contract and Graphify adapter to retain relation, source, evidence, and confidence; add strict versioned schemas and negative validation tests for unknown fields, secrets, machine paths, cycles, dangling edges, and invalid fingerprints.

## 2. Markdown map safe round trip

- [x] Read and validate one canonical managed nested-list map with stable Obsidian block IDs; interpret headings or ordinary lists only as read-only adoption candidates; implement deterministic serialization, revisions, persistence, and Visual Edit Plan preview/apply with source-hash locking, affected paths, actor, plan fingerprint, transition token, byte-preserved surrounding prose, optimistic-conflict rejection, replay safety, and cross-Project isolation.

## 3. Portable projections and Graphify-assisted structure

- [x] Derive a deterministic primary tree and secondary cross-links from explicit structure, reviewed Work-OS/knowledge relations, and explicitly accepted Graphify/model suggestions; retain Graphify provenance and confidence; implement bounded dependency-free layout plus textual, Mermaid mindmap, and Obsidian core Canvas projections with stable ordering, source links, node/depth limits, truncation diagnostics, golden fixtures, and no change to the existing project workflow Canvas contract.

## 4. Ask Mate outline-first minimum experience

- [x] Add the Obsidian Ask Mate ItemView and commands for active note, selection, supported Canvas, Mind Map Document, and Project Context; provide `Understand this` and `Make or revise a map` through a keyboard-operable outline with live deterministic preview, Graphify-assisted optional relation suggestions, ambiguity clarification, selectable structural diff, warnings, explicit confirmation, safe model/degraded mode, view restoration, mobile bounds, and tests proving no pre-confirmation write or plugin-private canonical state. Direct node dragging and ecosystem mind-map editors remain optional follow-on adapters.

## 5. OBC to local Work-OS loop

- [x] Implement Problem Observation contracts, fingerprints, recurrence, lifecycle, verification, redaction, bounded persistence, and provider-neutral normalization; route first-party OBC through one shared engine, replace or redirect duplicate `vault.lint` behavior, and deliver `Report or fix a problem` with deduplicated local Issue Change Plan preview/apply through Project Operations only, including duplicate-issue, dismissal/reopen, failed-provider, secret-leak, and idempotency tests.

## 6. Optional plugin diagnostics and Project Hub trace

- [x] Define a versioned typed, read-only Obsidian plugin diagnostic adapter contract and Host Capability enforcement for Project Context, Work Run, Assignment Plan, Capability Grant, allowlist, timeout, schema, and redaction; implement one low-risk open-source reference adapter without arbitrary command execution; extend Project Hub, Bases, Canvas, and textual projections with map freshness, provider health, observations, linked issues, Work Runs, verification, and end-to-end traceability while keeping every owning domain authoritative.

## 7. User-approved upstream Issue

- [x] Implement immutable `local_only` and `submit_issue` External Contribution Plans with provider-neutral repository preflight, GitHub first, editable secret-safe target/title/body/evidence/labels/redactions/warnings, linked reviewed local work, explicit per-run approval, pending/success receipts, duplicate prevention, cancellation at every step, idempotent replay, outcome-unknown reconciliation, Project Hub links, and no inferred remote consent.

## 8. Verified draft pull request and release gate

- [x] Implement `prepare_pull_request` with governed forge binding, isolated worktree, stable base/head facts, bounded diff, regression test evidence, secret and permission checks, Issue fallback, separate branch/fork push confirmation, draft-only PR creation, separately previewed ready-for-review transition, replay-safe receipts, and no merge; complete Canvas supported-subset adoption, migration/doctor/privacy/GPL documentation, dependency and transitive-license audit, accessibility/mobile/round-trip/clean-vault acceptance, cross-host tests, bundles, and experimental-release gates.
