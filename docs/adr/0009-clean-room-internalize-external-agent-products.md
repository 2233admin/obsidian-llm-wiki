# Clean-room Internalize External Agent Products

LLM Wiki may study external Agent products as Sources and internalize product mechanisms only when those mechanisms are independently expressed through LLM Wiki's own domain model, contracts, storage formats, terminology, tests, and user experience. Product research SHALL record the canonical Source URL, inspected commit or version, license, observed behavior, and resulting LLM Wiki requirement before implementation begins.

For `EXXETA/exxperts`, the inspected product layer is governed by PolyForm Noncommercial 1.0.0 at commit `035594aad68db78ef92899578b2e4839343f53d9`. LLM Wiki will not copy or adapt its product code, prompts, tests, UI text, styles, icons, screenshots, fixtures, or assets, and will not introduce an EXXETA product dependency. Engineers implementing `add-governed-agent-rooms-and-dreamtime` must work from the reviewed OpenSpec requirements and existing LLM Wiki contracts rather than from EXXETA source files.

The mechanisms selected for independent implementation are persistent Project-rooted Agent Rooms, proposal-and-approval working memory, read-only Agent consultation, Work Run-based delegation, governed host capability connectors, and usage observability. EXXETA-specific frontend structure, runtime packaging, personas, prompt wording, naming, and asset treatment are outside scope.

For `Radiant303/SpringNote`, LLM Wiki inspected public commit `9c062d91a8cf93bfb51406d9475417091437c625` on 2026-07-15 under AGPL-3.0. The repository is registered as Source `src_2af75360003e`. LLM Wiki will not copy or adapt SpringNote code, prompts, tests, UI text, styles, icons, screenshots, fixtures, or assets and will not introduce SpringNote as a dependency. The independently selected product mechanisms are daily/weekly/monthly reflection, memory-oriented retrieval, and usage/activity observability. In LLM Wiki these mechanisms must be expressed as Project-scoped Work Runs, deterministic Context Envelopes, proposal-only Dream Time operations, explicit approval into immutable Memory Revisions, and append-only Usage Events. SpringNote's desktop shell, visual treatment, gamification, report wording, provider UI, and implementation structure remain outside scope.

For `vectorize-io/hindsight`, LLM Wiki inspected public commit
`5ab6bdc9b63b76ba644124bf65a0fb18c72db7d9` under MIT and registered Source
`src_ef1d62b18b98`. The selected mechanism is only a provider-neutral,
read-only Knowledge Adapter that maps LLM Wiki search to a configured Hindsight
memory-bank recall endpoint. Hindsight retain, reflect, learned mental models,
storage, prompts, SDK code, tests, UI, and assets are not copied or made LLM
Wiki memory authority. Any credential remains a device-local Secret Reference,
and durable LLM Wiki knowledge still passes through its own Dream Time, Memory
Governance, and Promotion boundaries.

If a future implementation needs runtime functionality similar to the separately marked MIT-derived runtime directory, dependency selection must evaluate the MIT upstream independently and record a new dependency/license decision. This decision does not grant permission to copy any EXXETA product-layer material or to bypass Source Registration, Memory Governance, Promotion Policy, Operation Write Policy, or third-party notice obligations.
