# ADR 0011: Generate Markdown Views by Default and Keep Obsidian Views Optional

## Status

Accepted

## Context

LLMwiki vault state must remain useful without Obsidian-specific features. Markdown views provide accessible read models for agents, humans, CI, and plain editors. Obsidian Bases and Canvas can improve navigation, but they should not become required canonical state.

## Decision

Markdown views are generated by default. Obsidian-specific views are optional enhancements.

Rules:

- `views/dashboard.md` and `views/source-map.md` are core Markdown read models.
- Successful `vault apply-ingest-output` refreshes Markdown views.
- `vault update-views <vault>` provides explicit manual refresh.
- `.base` and `.canvas` outputs require `--obsidian` or configuration.
- Normal vault lint reports missing Obsidian views as info.
- Release-check does not fail on missing Obsidian views unless a release profile requires them.
- Canvas outputs must have a Markdown companion table.

## Consequences

- The vault works in any Markdown environment.
- Obsidian polish can be added without blocking the audit loop.
- Generated visual outputs cannot hide canonical state.
