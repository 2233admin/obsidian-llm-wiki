LLM Wiki Bridge is a headless-first MCP server that turns wikilinks, aliases, and frontmatter into a concept graph your agent can query. Six personas ship with it. Obsidian optional.

LLM Wiki Bridge 是一个 headless-first 的 MCP 服务器，把 wikilink、别名和 frontmatter 编译成一张概念图， 让你的 agent 直接调用查询。内置六个 persona，Obsidian 可选。

The problem this solves

它解决的问题

You have 500 notes. You wrote half of them last year. You do not remember what is in them.

你有 500 篇笔记。一半是去年写的。你已经想不起来里面写了什么。

Your agent cannot read your vault. It guesses from its context window and hallucinates citations.

你的 agent 读不到你的 vault。它只能在自己的上下文里猜，然后编造引用。

Every morning you spend 20 minutes re-finding what you already knew.

每天早上你要花 20 分钟，把昨天已经知道的东西重新找出来。

Three axioms that shape it

三条基本约束

  1. Filesystem is the floor.

    文件系统是底线。

    If a feature needs more than a directory of markdown files, it is optional -- not default. Obsidian, memU, gitnexus are all opt-in.

    任何功能如果要求超过"一个 markdown 文件夹"，它就是可选的，不是默认。Obsidian、memU、gitnexus 全部 opt-in。

  2. Compile once, query many times.

    编译一次，多次查询。

    554 notes to 2,507 edges in under 3 seconds. Every subsequent vault.search, vault.backlinks, query.unified reuses that graph.

    554 篇笔记、2,507 条边，3 秒内编译完。之后每次 vault.search、vault.backlinks、query.unified 都复用这张图。

  3. Agents cite, don't guess.

    Agent 引用，不瞎猜。

    The agent calls vault.read on the cited notes and answers with evidence. It does not paraphrase a grep hit-list from memory.

    Agent 对引用的笔记调用 vault.read，拿到原文再作答。它不会看一眼 grep 结果就开始编。

Six personas, one MCP surface

六个 persona，一套 MCP 接口

每个 persona 都是同一套 40 操作 MCP 接口上的一段有主见的提示词。一个负责检索与引用，一个负责找孤儿笔记，一个回答"我 4 月 3 日在想什么"。

Architecture at a glance

架构一览

+-------------------------------------------------------+
| L4  Agent Scheduler                                   |
|     compile | emerge | reconcile | prune | challenge  |
+-------------------------------------------------------+
| L3  Compiler Pipeline                                 |
|     link discovery -> graph build -> evaluation       |
+-------------------------------------------------------+
| L2  Adapter Registry                                  |
|     filesystem (required) | obsidian | memU | gitnexus|
+-------------------------------------------------------+
| L1  MCP Server                                        |
|     stdio transport, 40 ops across 5 namespaces       |
+-------------------------------------------------------+

The filesystem adapter is the floor. Obsidian, memU, gitnexus are optional and degrade gracefully when absent. For the full request lifecycle and extension points, see the Architecture wiki page.

filesystem adapter 是底座。Obsidian、memU、gitnexus 都是可选的，不在时优雅降级。 完整请求生命周期与扩展点见 Architecture wiki 页。

Honest limits

诚实的边界

  • It does not understand code in your notes -- it indexes text, wikilinks, and structure. 它不理解笔记里的代码 -- 只索引文本、wikilink、结构。
  • It does not sync bidirectionally with Obsidian in real time -- the WebSocket adapter requires Obsidian to be running. 它不与 Obsidian 实时双向同步 -- WebSocket adapter 需要 Obsidian 在运行。
  • It does not replace a vector database for semantic similarity at scale -- enable the optional memU adapter if you need that. 它在大规模语义相似度上不能替代向量数据库 -- 需要时请启用可选的 memU adapter。
