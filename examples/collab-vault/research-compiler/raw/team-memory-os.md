# Team Memory OS

A research team needs a shared memory loop, not a private assistant. Raw source material should be cheap to capture, compiled summaries should be easy to inspect, and durable conclusions should pass review before they become team truth.

The operating path is:

- capture papers, repo notes, clippings, and datasets in `raw/`
- compile summaries and concepts into `wiki/`
- answer questions from cited notes
- file candidate answers in `00-Inbox/AI-Output/`
- promote reviewed knowledge into `20-Decisions/`, `30-Architecture/`, or `40-Runbooks/`

The useful roles are work roles: librarian for retrieval, architect for structure, curator for quality, janitor for cleanup, and compiler for raw-to-wiki conversion.
